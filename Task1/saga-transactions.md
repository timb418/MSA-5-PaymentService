# Реестр транзакций Saga-оркестрации — OrchestrPay

## Описание процесса

Saga охватывает полный цикл обработки платежа: от инициации пользователем до перевода средств контрагенту, включая антифрод-проверку с тремя возможными исходами (разрешение, запрет, ручная проверка) и все компенсационные сценарии.

## Таблица транзакций

| Транзакция | Описание | Тип | Компенсация |
|---|---|---|---|
| `CREATE_PAYMENT` | Создание записи о платеже в системе, инициализация платёжной сессии с уникальным идентификатором | компенсируемая | `CANCEL_PAYMENT` |
| `DEBIT_ACCOUNT` | Списание средств со счёта плательщика и резервирование суммы для проведения операции | компенсируемая | `CREDIT_ACCOUNT` |
| `FRAUD_CHECK` | Отправка запроса в антифрод-сервис (FraudCheck Service) с поддержкой retry при таймауте; возвращает: разрешение, запрет или ручная проверка | повторяемая | — |
| `WAIT_MANUAL_REVIEW` | Ожидание решения оператора при ручной антифрод-проверке; максимальное время ожидания — 20 минут; по истечении cut-off-time транзакция автоматически считается разрешённой | повторяемая | — |
| `TRANSFER_FUNDS` | Перевод средств на счёт контрагента (продавца / получателя) — **точка невозврата (pivot)**: после успешного перевода операция не может быть отменена в рамках текущей Saga | необратимая | — |
| `UPDATE_PAYMENT_STATUS` | Фиксация итогового статуса платежа (COMPLETED / FAILED / REFUNDED) в базе данных PostgreSQL | повторяемая | — |
| `NOTIFY_CLIENT_SUCCESS` | Отправка уведомления клиенту об успешном проведении платежа через Notification Service | повторяемая | — |
| `CANCEL_PAYMENT` | Отмена и аннулирование записи о платеже (компенсирующая транзакция для `CREATE_PAYMENT`) | повторяемая | — |
| `CREDIT_ACCOUNT` | Возврат зарезервированных/списанных средств на счёт плательщика (компенсирующая транзакция для `DEBIT_ACCOUNT`) | повторяемая | — |
| `NOTIFY_CLIENT_FAILURE` | Отправка уведомления клиенту об отклонении платежа или успешном возврате средств через Notification Service | повторяемая | — |
| `NOTIFY_SECURITY` | Уведомление службы безопасности о подозрительной транзакции для последующего анализа (только при обнаружении мошенничества) | повторяемая | — |

## Пояснения к типам транзакций

- **Компенсируемая** — транзакция, для которой определена компенсирующая операция. Выполняется до точки невозврата. При сбое на любом последующем шаге запускаются компенсации в обратном порядке.
- **Необратимая (pivot)** — точка невозврата Saga. После её успешного выполнения компенсации не применяются; все последующие шаги должны в итоге завершиться успехом.
- **Повторяемая** — транзакция, которая гарантированно завершится успехом при повторных попытках (идемпотентна). Используется как после pivot-точки, так и в качестве компенсирующих шагов.

## Сценарии выполнения Saga

### Успешный сценарий
`CREATE_PAYMENT` → `DEBIT_ACCOUNT` → `FRAUD_CHECK` → `TRANSFER_FUNDS` → `UPDATE_PAYMENT_STATUS` → `NOTIFY_CLIENT_SUCCESS`

### Сценарий с ручной проверкой (approved)
`CREATE_PAYMENT` → `DEBIT_ACCOUNT` → `FRAUD_CHECK` → `WAIT_MANUAL_REVIEW` → `TRANSFER_FUNDS` → `UPDATE_PAYMENT_STATUS` → `NOTIFY_CLIENT_SUCCESS`

### Сценарий с ручной проверкой (cut-off истёк)
`CREATE_PAYMENT` → `DEBIT_ACCOUNT` → `FRAUD_CHECK` → `WAIT_MANUAL_REVIEW` *(timeout 20 мин)* → `TRANSFER_FUNDS` → `UPDATE_PAYMENT_STATUS` → `NOTIFY_CLIENT_SUCCESS`

### Сценарий с отклонением антифродом
`CREATE_PAYMENT` → `DEBIT_ACCOUNT` → `FRAUD_CHECK` *(запрет)* → `CREDIT_ACCOUNT` → `CANCEL_PAYMENT` → `NOTIFY_CLIENT_FAILURE`

### Сценарий с обнаружением мошенничества
`CREATE_PAYMENT` → `DEBIT_ACCOUNT` → `FRAUD_CHECK` *(мошенничество)* → `CREDIT_ACCOUNT` → `CANCEL_PAYMENT` → `NOTIFY_CLIENT_FAILURE` → `NOTIFY_SECURITY`
